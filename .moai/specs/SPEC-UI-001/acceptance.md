# SPEC-UI-001: 인수 테스트 기준

> 본 문서는 SPEC-UI-001의 각 EARS 요구사항이 충족되었는지 검증하기 위한 Given-When-Then 인수 시나리오와 성능·품질 게이트를 정의한다.

## 인수 시나리오

### M1. 포즈 추정 파이프라인

**시나리오 1: 정상 포즈 추정**
- Given: 활성화된 운동 세션에서 사용자 전신이 카메라 프레임 안에 정상적으로 보이는 상태
- When: 카메라 프레임이 PoseEstimationService로 도착함
- Then: 시스템은 33개 관절 키포인트를 온디바이스에서 추출하고, 단일 프레임 추론을 100ms 이내에 완료한다

**시나리오 2: 저가시성 키포인트 필터링 (에지 케이스)**
- Given: 사용자의 하반신 일부가 프레임 밖에 있어 일부 키포인트의 visibility가 임계값 미만인 상태
- When: 해당 프레임의 키포인트가 각도 분석 단계로 전달됨
- Then: 시스템은 visibility 임계값 미만인 관절을 각도 계산에서 제외하고, 신뢰 가능한 관절만으로 계산을 수행한다 (저신뢰 관절로 인한 오탐 발생 안 함)

### M2. 관절 각도 분석

**시나리오 3: 스쿼트 폼 오류 감지**
- Given: 선택된 운동이 스쿼트이고 유효한 무릎 키포인트가 추출된 상태
- When: 계산된 무릎 각도가 임계 범위(80~120도)를 벗어남 (예: 60도)
- Then: 시스템은 해당 상태를 폼 오류로 분류하고 오류 관절(무릎)을 식별한다

**시나리오 4: 데드리프트 척추 정렬 오류**
- Given: 선택된 운동이 데드리프트이고 유효한 척추 관련 키포인트가 추출된 상태
- When: 계산된 척추 각도가 중립 정렬 임계를 벗어남 (등이 둥글게 굽음)
- Then: 시스템은 해당 상태를 폼 오류로 분류하고 척추 정렬 오류로 식별한다

### M3. 실시간 피드백 시스템

**시나리오 5: 피드백 표시**
- Given: M2에서 폼 오류가 감지된 상태
- When: 오류 분류 결과가 FeedbackGenerator로 전달됨
- Then: 시스템은 카메라 화면 위에 교정 텍스트 메시지(예: "무릎을 더 펴세요")를 표시하고, expo-speech로 동일 내용의 음성 피드백을 재생하며, 오류 감지부터 표시까지 500ms 이내에 완료한다

**시나리오 6: 디바운스 — 중복 음성 억제 (에지 케이스)**
- Given: 동일한 무릎 폼 오류에 대한 음성 피드백이 직전에 재생된 상태
- When: 동일 오류가 디바운스 구간(2초) 이내에 다시 감지됨
- Then: 시스템은 중복 음성을 재생하지 않는다 (텍스트 표시는 유지될 수 있으나 음성 알림 피로를 방지)

### M4. 스켈레톤 오버레이

**시나리오 7: 실시간 스켈레톤 렌더링**
- Given: 운동 세션이 진행 중이고 키포인트가 정상 추출되는 상태
- When: 매 프레임 키포인트가 CameraOverlay로 전달됨
- Then: 시스템은 카메라 피드 위에 33관절 골격 연결선을 30 FPS 이상으로 렌더링하며, 폼 오류가 활성 상태일 때 문제 관절을 경고 색상(적색)으로 강조한다

### M5. 카메라 배치 가이드

**시나리오 8: 진입 시 가이드 표시**
- Given: 사용자가 운동 화면(CameraScreen)에 진입하는 시점
- When: 화면 진입 이벤트가 발생함
- Then: 시스템은 최적 카메라 위치(측면 45도, 골반 높이) 오버레이 가이드를 표시한다

**시나리오 9: 부적절한 앵글 경고**
- Given: 카메라가 정면 또는 부적절한 높이에 배치되어 추정 앵글이 허용 범위를 벗어난 상태
- When: 앵글 적정성 판정 로직이 실행됨
- Then: 시스템은 재배치 경고 메시지를 표시한다 (가능하면 가시성 점수 기반 적정성 점수도 함께 제공)

## 성능 게이트 (Pass/Fail 기준)

| 항목 | 기준 | 판정 |
|------|------|------|
| 프레임당 추론 지연 | < 100ms | Pass/Fail |
| 스켈레톤 렌더 FPS | >= 30 | Pass/Fail |
| 피드백 응답 (감지→표시) | < 500ms | Pass/Fail |
| 앱 메모리 사용량 | < 150MB | Pass/Fail |
| 시간당 배터리 영향 | < 5% | Pass/Fail |

> 성능 측정은 통합 프로파일링(시나리오 7 + 1 + 5 동시 동작)으로 단일 디바이스에서 수행하며, MVP 최소 사양 기기에서 게이트를 통과해야 한다.

## Definition of Done

- M1~M5 전체 EARS 요구사항에 대응하는 인수 시나리오(1~9)가 모두 Pass
- 에지 케이스(시나리오 2, 6) 검증 완료
- 성능 게이트 전 항목 통과 (최소 사양 기기 기준)
- 품질 게이트 전 항목 통과
- 온디바이스 추론으로 영상·키포인트 외부 미전송 확인
- 2D 카메라 한계 면책 고지 UI 포함
